Part 1: Localhost vs Production

Ex1.1: Phát hiện anti-patterns
5 vấn đề trong file app.py
- API đang hardcode trong code
- Config đang được đặt cả trong code, nên đặt ở env
- Không sử dụng Logging mà print 
- Chưa có health check api, để server biết khi agent crash, nó restart lại
- Port đang đặt cố định, chưa được đặt trong .env

Ex1.2: Chạy basic version
curl http://localhost:8000/ask -X POST \
  -H "Content-Type: application/json" \
  -d '{"question": "Hello"}'

Đã chạy tuy nhiên chưa đủ để production ready vì chưa có:
- API key / JWT
- Pydantic validation
- Exception handling
- Health endpoint
- Env variables
- Logging
- Rate limiting

Ex1.3: So sánh với advanced version

| Feature | Basic | Advanced | Tại sao quan trọng? |
|---------|-------|----------|---------------------|
| Config | Hardcode | Env vars | Tách cấu hình ra khỏi source code, tránh lộ secret, dễ thay đổi giữa các phiên bản dev/product |
| Health check | /heath endpoint|  | Kiểm tra service còn sống, để restart khi lỗi |
| Logging | print() | JSON | Dễ theo dõi lỗi, phân tích log và tích hợp với monitoring systems |
| Shutdown | Đột ngột | Graceful | Cho phép xử lý request đang chạy, đóng kết nối sạch sẽ, tránh mất dữ liệu |

Part 2: Docker Containerization

Exercise 2.1: Dockerfile cơ bản
Đọc Dockerfile và trả lời:
Base image là gì? Base image là image nền ban đầu để tạo container. Nó chứa hệ điều hành Linux tối thiểu, python 3.11, pip, các hệ thống cơ bản
Working directory là gì? Tạo thư mục làm việc hiện tại, tất cả các lệnh sau đó sẽ chạy trong thư mục này
Tại sao COPY requirements.txt trước? Tận dụng Docker layer cache, mõi lần cài lại không cần phải cài lại dependencies.
CMD vs ENTRYPOINT khác nhau thế nào? CMD cung cấp lệnh mặc định có thể ghi đè; ENTRYPOINT định nghĩa lệnh chính của container khó bị thay đổi hơn.

Exercise 2.2: Build và run
IMg Size = 424MB

Sau khi sử dụng Multi-stage build thì size đã giảm xuống còn 56.6 MB, giảm gần 7 lần 
Stage 1 làm gì?

Stage 1 cài các build tools và Python dependencies cần thiết để chuẩn bị môi trường ứng dụng.

Stage 2 làm gì?

Stage 2 tạo image runtime cuối cùng chỉ chứa source code và packages cần để chạy ứng dụng.

Tại sao image nhỏ hơn?

Vì image cuối không chứa compiler, build tools và file tạm từ quá trình build nên dung lượng nhỏ hơn và an toàn hơn.

Services nào được start? Chúng communicate thế nào? 
Có tổng cộng 4 dịch vụ chính sẽ được khởi động:

nginx: Đóng vai trò là cổng chào (Reverse Proxy/Load Balancer). Đây là dịch vụ duy nhất mở cổng ra bên ngoài (Port 80 và 443).

agent: Chứa mã nguồn FastAPI xử lý logic AI. Nó chạy trên port nội bộ 8000.

redis: Lưu trữ dữ liệu tạm thời (cache) và quản lý giới hạn tần suất (rate limiting). Chạy trên port 6379.

qdrant: Cơ sở dữ liệu vector (Vector Database) để phục vụ tìm kiếm ngữ nghĩa (RAG). Chạy trên port 6333.
